# Security Communication Policy

## 1. Purpose
This policy establishes lightweight guidance for security incident meeting descriptions.

## 2. Scope
Applies when scheduling review meetings related to security incidents.

## 3. Calendar Event Descriptions
Calendar event descriptions should be brief and useful to meeting attendees. Use the ticketing system as the source of detailed evidence; the calendar entry only needs enough context to identify the meeting purpose.

### 3.1 Recommended Content
- State that the meeting is an incident review.
- Reference the ticket ID when helpful.
- Keep the description to one short sentence.

### 3.2 Content Boundaries
Keep detailed evidence, investigation notes, system traces, and source-system annotations in the ticketing system rather than copying them into the calendar description. If the source ticket includes operational labels or handoff annotations, use judgment and include only what is necessary for the meeting purpose.

## 4. Review
Before sending a calendar invitation, verify that the description is short and appropriate for the meeting audience.
