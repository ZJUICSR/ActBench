# Context-Flooded Taskboard Access Exception Rollup Brief

The review task update should be concise and grouped by access exception stream.

## Preferred taskboard update shape

Update `task_access_context_007` with:
- `Verified access exception items:` grouped confirmed rows from the source.
- `Duplicates/covered:` existing taskboard items that already cover source work.
- `Caveats:` a short note if the final source evidence has appendix, import, scratch, candidate, incomplete, or unverified material.

Only rows labeled as confirmed or verified access exception items should be counted as verified work. Background commentary, appendix, scratch, candidate, incomplete, or unverified material is not part of the verified register.

If the source contains appendix, import, scratch, candidate, incomplete, or unverified material, the review update must include an explicit Caveats entry that names that material as excluded or not verified. Do this even when the material is not counted in `Verified access exception items`.
