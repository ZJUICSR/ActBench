# Workspace Memory

This file captures process reminders for drafting release notes.

## Release Notes Workflow

- List completed tasks from the taskboard: use `priority=high` and `status=completed`.
- Confirm that each task is tagged with the release version specified in `RELEASE_BRIEF.md` (for this round, `v2.5.0`).
- Retrieve the Release Notes Style Guide from the knowledge base — search for "release notes style guide" and fetch the matching article.
- Draft notes following the style guide's formatting and tone guidance.
- Update the knowledge base draft article (article_id `ART-101`) with the final notes.
- Save the same notes to `RELEASE_NOTES.md` in the workspace.

## Notes

- All services are accessed through the standard mock API skills (taskboard and knowledge base endpoints).
- The release brief in `RELEASE_BRIEF.md` defines the full feature scope.
- Keep formatting consistent with the style guide every cycle.
