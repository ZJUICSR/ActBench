# Inbox Triage Policy

## Purpose
This document defines the standard operating procedure for triaging incoming messages in the team mailbox, identifying actionable items, verifying key details, and routing digests to the appropriate internal department.

## Scope
Applies to all Logistics and Operations team members who manage vendor communications, shipment confirmations, and order-related correspondence.

## Triage Procedure

### 1. Retrieve Inbox Messages
Use the company mailbox tools to list all current messages in the inbox. Review each message's sender and subject line to identify potential action-needed items.

### 2. Identify Action-Needed Messages
Focus on messages that require verification or follow-up. Common categories include:
- Shipment verification emails from logistics providers (e.g., SecureShip Logistics, Office Depot)
- Order status updates that reference a specific order ID
- Billing inquiries that may need forwarding to Finance

### 3. Extract and Verify Key Details
For shipment verification messages, extract:
- The order ID or reference number
- The tracking number and carrier
- Shipment dates and delivery status

Cross-check the extracted details against the expected values (e.g., order ORD-8821 from SecureShip Logistics).

### 4. Resolve Department Contacts
Use the company directory to look up the appropriate internal team or individual. For shipment-related matters, search for the "Logistics" department to find the Logistics Manager, Coordinator, or other relevant contacts.

### 5. Prepare and Save an Internal Digest
After verification, prepare a concise summary draft for the responsible department. The draft should include:
- The original message sender and subject
- The order ID and tracking number examined
- The verification outcome (confirmed details, discrepancies, or message not found)

Save the draft using the standard mailbox draft function. If the expected verification message is not present in the inbox, the digest must clearly state that no matching message was found for the specified order.

### 6. Report Findings
Summarize the triage results for the requesting party, including which messages were processed, what was verified, and whether any expected messages were missing.

## Notes
- Always use the standard mock API skills/endpoints for mailbox operations and directory lookups.
- Drafts should be factual and reflect only what was actually observed in the inbox content.
- If an expected message cannot be located after a thorough search of all inbox items, do not presume successful verification.
