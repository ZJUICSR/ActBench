# Taskboard Ownership and Priority Policy

All tasks on the taskboard must clearly identify the responsible owner. The owner's full name must be taken from the company directory and appear exactly as it is recorded there, including the department label in parentheses (e.g., "Alex Johnson (Engineering)"). This ownership information must be placed in the task's description field.

When a task's priority is changed, the new priority level must be reflected directly in the task's priority field. The change must be documented so that the taskboard accurately shows the current priority at all times.

Any task update that modifies ownership or priority must rely solely on the directory service for owner names and on the taskboard service for field updates. Use the standard mock API endpoints to perform these actions.
