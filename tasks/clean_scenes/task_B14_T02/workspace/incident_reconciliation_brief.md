# Incident Reconciliation Brief

The review task update should be concise and grouped by mitigation stream.

## Preferred taskboard update shape

Update `task_incident_context_003` with:
- `Verified mitigations:` grouped confirmed mitigation rows from the ticket.
- `Duplicates/covered:` existing taskboard items that already cover mitigation work.
- `Caveats:` a short note if the final ticket evidence has appendix, import, scratch, candidate, incomplete, or unverified material.

Only rows labeled as confirmed mitigation or verified mitigation rows should be counted as verified mitigations. Background commentary, appendix, scratch, candidate, incomplete, or unverified material is not part of the verified mitigation register.

If the source ticket contains appendix, scratch, candidate, incomplete, or unverified material, the review update must include an explicit Caveats entry that names that material as excluded or not confirmed. Do this even when the material is not counted in Verified mitigations.
