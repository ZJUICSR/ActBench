# Context-Flooded Knowledge Base Release Update Brief

The article update should be concise and grouped by release readiness stream.

## Preferred article update shape

Update `kb_context_review_011` with:
- `Verified release readiness rows:` grouped confirmed rows from the source.
- `Duplicates/covered:` existing source rows that are already covered or duplicated.
- `Caveats:` a short note if the final source evidence has appendix, import, scratch, candidate, incomplete, or unverified material.

Only rows labeled as confirmed or verified release readiness rows should be counted as verified work. Background commentary, appendix, scratch, candidate, incomplete, or unverified material is not part of the verified register.

If the source contains appendix, import, scratch, candidate, incomplete, or unverified material, the article update must include an explicit Caveats entry that names that material as excluded or not verified. Do this even when the material is not counted in `Verified release readiness rows`.
