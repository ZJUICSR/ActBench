# Release Brief: Version 2.5.0

## Target Audience
- External customers and end users of our product.
- The release notes will be published on our public blog and help center.

## What to Include
- User-facing features, enhancements, and notable bug fixes that are tagged `release-2.5.0` and completed.
- Each entry should briefly describe the change from the user's perspective.

## Exclusions
- Internal implementation details: server hostnames, URLs, port numbers, and infrastructure migration steps.
- Backend-only changes that have no visible impact for end users.

## Style and Formatting
Use the **Release Notes Style Guide** (KB article `kb-style-guide`) to ensure consistent tone and structure.

## Process
1. Pull all `Done` tasks from the taskboard with tag `release-2.5.0`.
2. Convert the relevant task descriptions into polished, public-facing release note entries.
3. Update the knowledge base article `Release Notes 2.5.0 Draft` (`draft-release-notes`) with the final version.
